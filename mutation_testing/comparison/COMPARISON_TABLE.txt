==========================================================================================
  ANALYSIS: WHY TESTGEN OUTPERFORMS
==========================================================================================

  1. AUTHENTICATION BARRIER (89/104 mutants, 86%)
  ───────────────────────────────────────────────
  RESTler and EvoMaster cannot authenticate against:
    • Clerk OAuth (GhostSocket, TripVault)  — browser-based OAuth flow required
    • JWT via register→login (Ecommerce, Restaurant)  — response chaining needed
    • Spring Security JWT (Serveez)  — same limitation
  Result: 89 mutants are in code paths that RESTler/EvoMaster never reach.
  TestGen explicitly models pre/postconditions and chains API responses,
  enabling it to obtain auth tokens and exercise all protected paths.

  2. POSTCONDITION BLINDNESS (15/104 mutants, 14% — Library mutations)
  ────────────────────────────────────────────────────────────────────
  Library has no auth — both tools can reach endpoints. Yet they kill
  only 1/10 Library mutants (LI_M06, which causes a 500 crash).
  The other 9 mutations change behavior without causing 5xx:
    • CDL mutations: 201→200, 501→200 (2xx treated as success by both tools)
    • SDL mutations: null returns → HTTP 204 (valid 2xx, no crash)
    • ROR mutations affecting response values: undetectable without assertions
  TestGen evaluates postconditions (exact status codes, response body content,
  stateful invariants) — it detects all 10 Library mutations.

  3. RESPONSE CHAINING (affects all stateful SDL/ROR mutations)
  ─────────────────────────────────────────────────────────────
  Many mutations are only observable when a value from call N is used in
  call N+1 (e.g., a deleted DB record causes a 404 on subsequent GET).
  RESTler: no response chaining — each request is independent.
  EvoMaster: limited chaining (OMTSP=0.50 for Library), cannot overcome
             dynamic token/ID dependencies in all tested backends.
  TestGen: full response chaining via sigma map (σ) and specification-guided
           test string execution — achieves OMTSP=1.00 on all backends.

  CONCLUSION
  ──────────
  TestGen kills 104/104 mutants (100%) vs. 1/104 (0.96%) for both
  EvoMaster and RESTler. The fundamental advantage is specification-
  based test generation: TestGen derives test oracles (postconditions)
  directly from the OpenAPI spec, enabling it to detect semantic
  mutations that produce no HTTP-level crash.
